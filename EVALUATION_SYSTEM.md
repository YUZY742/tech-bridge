# TECH-BRIDGE 人材評価システム

## 評価の基本方針

TECH-BRIDGEでは、「実証された能力」を評価するため、以下の3つの軸で人材を評価します：

### 1. 技術力（Technical Score）40%

**評価項目：**
- **ポートフォリオの質と量**（30点満点）
  - GitHubリポジトリの数と質
  - 設計図面（DR）の有無
  - 実験データの記録
  - 活動報告書の充実度
- **技術スタックの多様性**（20点満点）
  - 使用言語の種類
  - CADソフトの習熟度
  - 実験設備の活用
- **貢献の質**（25点満点）
  - 具体的な貢献内容の記録
  - 技術的な深さ

**計算式：**
```
技術力スコア = min(100, 
  min(30, ポートフォリオ項目数 × 5) +
  min(20, 技術スタック種類数 × 4) +
  min(25, 貢献数 × 2)
)
```

### 2. 継続性（Grit Score）35%

**評価項目：**
- **活動日数**（40点満点）
  - 実際に活動した日数
  - 継続的な参加を示す
- **プロジェクト継続期間**（30点満点）
  - 最初の活動から現在までの期間
  - 長期的なコミットメントを示す
- **活動頻度**（30点満点）
  - 1日あたりの活動ログ数
  - 密度の高い活動を示す

**計算式：**
```
継続性スコア = min(100,
  min(40, 活動日数 × 2) +
  min(30, 継続期間（月） × 10) +
  min(30, 活動頻度（回/日） × 5)
)
```

### 3. 貢献度（Contribution Score）25%

**評価項目：**
- **総貢献数**（50点満点）
  - 活動ログに記録された貢献の総数
- **詳細な貢献**（30点満点）
  - 20文字以上の詳細な貢献記録
  - 具体的な成果を示す
- **受けた支援**（20点満点）
  - 企業からの支援件数
  - 価値が認められた証拠

**計算式：**
```
貢献度スコア = min(100,
  min(50, 総貢献数 × 5) +
  min(30, 詳細貢献数 × 3) +
  min(20, 受けた支援数 × 2)
)
```

## 総合評価

**総合スコア（Overall Score）：**
```
総合スコア = 技術力 × 0.4 + 継続性 × 0.35 + 貢献度 × 0.25
```

**評価ランク：**
- **80点以上**: 優秀 - 企業から高く評価される可能性が高い
- **60-79点**: 良好 - 実証された能力を持つ
- **40-59点**: 普通 - さらなる活動が推奨される
- **40点未満**: 要改善 - 活動ログやポートフォリオの充実が必要

## 評価の特徴

### 1. 実証ベースの評価
- 主観的な自己PRではなく、実際の活動データに基づく
- 活動ログ、貢献記録、ポートフォリオなど、検証可能な情報のみを使用

### 2. 非認知能力の評価
- 「Grit（やり抜く力）」を継続性スコアで可視化
- 長期的なプロジェクトへのコミットメントを評価

### 3. 多面的な評価
- 技術力だけでなく、継続性と貢献度も重視
- バランスの取れた人材評価を実現

### 4. 透明性
- 評価基準が明確
- 学生自身がスコアを確認し、改善点を把握できる

## 企業側の活用方法

### 1. スクリーニング
- 総合スコアで初期選考
- 特定のスコア（例：技術力80点以上）でフィルタリング

### 2. 詳細評価
- 各スコアの内訳を確認
- 活動ログやポートフォリオを直接閲覧

### 3. マッチング
- 自社の技術基準に合わせて評価
- 継続性が高い学生を優先（長期プロジェクト向け）
- 技術力が高い学生を優先（即戦力向け）

## 今後の改善点

### 現状の課題
- **GitHub依存**: ポートフォリオ評価がGitHubリポジトリに依存している
- **定量的評価中心**: 定性的な評価要素が不足
- **多様性不足**: 評価軸が限定的

### 短期（現段階）
- ✅ 基本的な評価スコアの計算
- ✅ ダッシュボードでの可視化
- ⏳ 企業側からの評価フィードバック機能
- ⏳ 多角的評価の基盤構築

### 中期（1-3ヶ月）
- ポートフォリオの自動分析（GitHub API連携）
- 活動ログの品質評価（AIによる分析）
- 同世代との比較機能
- **多角的評価軸の追加**:
  - 協調性・チームワーク評価
  - 問題解決能力評価
  - 学習・成長能力評価
  - 外部評価（企業、メンター、ピアレビュー）
- 設計図面・実験データの質評価

### 長期（6-12ヶ月）
- 機械学習による予測モデル
- 企業の採用実績との相関分析
- 評価基準の最適化
- AI・機械学習の活用:
  - 自然言語処理による活動ログ分析
  - コード品質の自動評価

- 動的評価システム（リアルタイム更新）

## 注意事項

1. **評価は相対的**
   - スコアは絶対的なものではなく、参考指標として使用
   - 企業は必ず活動ログやポートフォリオを直接確認すべき

2. **活動ログの重要性**
   - 定期的な活動ログの記録が評価の精度を高める
   - 詳細な貢献記録を推奨

3. **ポートフォリオの充実**
   - GitHubリポジトリ、設計図面、実験データなどの記録が重要
   - 質の高いポートフォリオが技術力スコアに大きく影響
